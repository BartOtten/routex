      View Source Routex.Extension.Interpolation.NonUniqError exception (Routex v1.1.0)

      Raised when a list of routes contains routes with the same path and verb.

      [%Route{
  path: "/foo"
  verb: :get},
%Route{
  path: "/foo"
  verb: :post}, # <-- different
%Route{
  path: "/foo"
  verb: :get} # <-- duplicate
]

      Solution: use a combination of interpolated attributes that form a unique set.
