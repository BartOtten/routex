Phoenix Routes Extension Framework
v0.3.0-alpha.2

      View Source API Reference Phoenix Routes Extension Framework v0.3.0-alpha.2

      Modules

      Routex.Attrs

      Provides an interface to access and update Routex attributes.

      Routex.Backend

      use Routex.Backend

      When use'd this module generates a Routext backend module and a configuration struct using the configure/2 callbacks of the extensions provided in opts.

      Routex.Branching

      Provides a set of functions to build branched variants of macro's

      Routex.Dev

      Provides functions to aid during development

      Routex.Extension

      Specification for composable Routex extensions.

      Routex.Extension.AlternativeGetters

      Creates helper functions to get a list of alternative slugs and their Routex attributes by providing the function a binary url.

      Routex.Extension.Alternatives

      Creates alternative routes based on branches configured in a Routex backend module. Branches can be nested and each branch can provide Routex.Attrs to be shared with other extensions.

      Routex.Extension.Alternatives.Branch.Flat

      Struct for flattened branch

      Routex.Extension.Alternatives.Branch.Nested

      Struct for branch with optionally nested branches

      Routex.Extension.Alternatives.Config

      Module to create and validate a Config struct

      Routex.Extension.Alternatives.Exceptions
      Routex.Extension.Alternatives.Exceptions.AttrsMismatchError

      Raised when the custom attributes of branches do not have the same keys.

      Routex.Extension.Alternatives.Exceptions.MissingRootSlugError

      Raised when the branch map does not start with the root branch "/".

      Routex.Extension.Assigns

      Extracts Routex.Attrs from the route and makes them available in components and controllers with the @ assigns operator (optionally under a namespace).

      Routex.Extension.AttrGetters

      Creates helper functions to get the Routex.Attrs given a binary url or a list of path segments. This way the attributes for route can be lazily loaded.

      Routex.Extension.Cloak

      Transforms routes to be unrecognizable.

      Routex.Extension.Interpolation

      A route may be defined with a routes Routex.Attrs interpolated into it. These interpolations are specified using the usual #{variable} interpolation syntax.

      Routex.Extension.Interpolation.NonUniqError

      Raised when a list of routes contains routes with the same path and verb.

      Routex.Extension.RouteHelpers

      Provides route helpers with support for automatic selecting alternatives routes. The helpers can be used to override Phoenix' defaults as they are a drop-in replacements.

      Routex.Extension.Translations

      Enables users to enter URLs using localized terms which can enhance user engagement and content relevance.

      Routex.Extension.VerifiedRoutes

      Supports the use of unmodified route paths in controllers and templates while using transformed and/or branching paths with compile-time verification and dynamic runtime behavior.

      Routex.Matchable

      Matchables are an essential part of Routex. They are used to match run time routes with compile time routes.

      Routex.Processing

      This module provides everything needed to process Phoenix routes. It executes the transform callbacks from extensions to transform Phoenix.Router.Route structs and create_helpers callbacks to create one unified Helper module.

      Routex.Route

      Function for working with Routex augmented Phoenix Routes

      Routex.Router

      Provides macro (callbacks) to alter route definition before compilation.

      Routex.Test.Fixtures
      Routex.Test.Fixtures.Assigns
      Routex.Test.Support.Gettext
      Routex.Utils

      Provides an interface to functions which can be used in extensions.
